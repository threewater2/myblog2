正则表达式基础

2019.03.24 山水 Java

语法

单个字符

  1. 普通字符：1，2，0，水 ，z，A等
  2. 特殊字符：一般以\开头如：\n，\t 等
  3. 特殊字符和普通字符，都可以用八进制和十六进制表示，不过一般用的少
  4. \ 本身，如果要表示它，必须用两个：\
  5. 元字符：有特殊意义的字符，如：. * ? + 要表示这些字符本身，必须加上\

. 号可以匹配除换行符以外的任意字符，如果在单行匹配模式下，他也可以匹配换行符。
指定单行匹配模式。

  1. 在正则表达式中指定：（？s)a.b
  2. 在Java Api 中指定Pattern.DOTALL

字符组

字符组是用 [] 括起来的字符：如：[123] ，[0-9] ，[0-9a-z] 。排除0-9之间的字符[^0-9]。在字符组中 . 和* 就表示它本身。
有一些预定义的字符组：

  1. \d =[0-9] 与之对应的非为 \D
  2. \w = [0-9a-z_A-Z] \W
  3. \s = [\t\n\f\r] \S

量词

量词是指出现指定次数的字符组

    • ：表示前面的字符可以一次或者多次出现 ab{1,}
    • ：前面字符0次或者多次出现 ab{0,}
  3. ？ ：前面的字符出现一次或者零次 ab{0,1}
  4. ab{0，10} 出现0次到10次，ab{3}，必须出现三次

量词默认匹配方式贪婪的，碰到可能结束的地方不会提前结束，会一直匹配到最后一个
如果想要让他变的懒惰可以在后面加上？ 如：abccccb 对于a.?b ， 他有两次匹配。但对于 a.b 它只有一次匹配

分组

分组就是不可分割的组如：（ab）表示一组 （ab|234|er) 表示匹配其中的一组。

特殊边界匹配

  1. ^ 匹配字符串的开始，如^ab 表示字符串必须以ab开始
  2. $表示字符串必须以什么结束
  3. 以上两种默认都是单行匹配模式，也就是只匹配一行。可以指定其为多行匹配模式，这样对于多行他就会有多个结果匹配，（?m)ab$ 可以指定多行匹配模式或者使用Pattern.MULTILINE。
  4. \A 不管什么模式，匹配的总是整个字符串的开始边界。
  5. \Z 和 \z 不管什么模式，匹配的是整个字符串的结束边界，而\Z 匹配的是换行符之前的边界，而\z 匹配的总是结束边界。如：对于ab\nab ，ab\Z 和ab \z 都会匹配，对于ab\nab\n ， ab\z 不会匹配。
  6. \b 表示单词匹配， 如 我\b 可以匹配：它 我 它，不能匹配 是我是

环视边界匹配

  1. (?….) 右边的字符串能匹配指定的表达式，非是 (?!…)
  2. (?<=…) 左边的字符串匹配指定的表达式，非是(?!…)

把所有字符都视为普通字符

在\Q……\E 之间的字符都会视为不同字符
