cloneable接口

2019.02.24 山水 Java

Cloneable接口，cloneable接口一般与object的clone方法相关，clone是本地方法，如果类没有实现cloneable接口，那么clone方法会抛出异常，而且clone方法是proptected的，一般都要重写它，不然其它包无法使用它。

以上，说明该接口只是一个标志性接口，标志某个类是否可以被克隆。既然是克隆，那么就涉及到了深克隆，和浅克隆。object的clone方法默认实现是浅克隆。

浅克隆：

克隆的对象地址和原对象不同，但是对象里面的实例变量引用的地址相同，我们看一下是不是真的。

可以看到，浅克隆只是复制地址的引用。

深克隆：

就是克隆对象和原对象的地址不同，里面的实例变量的引用也不同。

深克隆只能自己实现，自己手动创建实例对象
